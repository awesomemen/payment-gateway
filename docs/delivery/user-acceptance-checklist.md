# 用户验收清单

本文档用于正式联调或用户验收阶段，不替代开发阶段的 `/acceptance/index.html` 工作台。

## 1. 验收前置条件

- 应用版本、镜像 tag、提交版本已确认
- MySQL、Redis、Nacos、RocketMQ、Seata、SkyWalking 已就绪
- 商户联调物料已确认：
  - 商户号
  - 签名密钥或证书引用方式
  - 回调地址
  - 下游路由信息
- 已确认验收环境使用的配置来源：
  - `application-local`
  - `Nacos`
  - `MySQL`
  - 混合模式

## 2. 基础连通性验收

- `GET /actuator/health` 返回 `UP`
- Nacos 控制台可访问，且网关服务已注册
- Seata 服务可发现
- RocketMQ producer / consumer 组配置已生效
- SkyWalking / Prometheus 至少能看到应用实例

## 3. 核心功能验收

- 支付创建成功
- 支付重复请求命中幂等
- 支付拒绝 / 失败 / 超时错误码符合约定
- 支付查询能够得到最终状态
- 支付回调能够推进状态
- 退款创建成功
- 退款查询能够得到最终状态
- 退款回调能够推进状态
- 统一交易查询能查到支付和退款记录
- 审计检索能查到请求日志和异常事件

## 4. 可靠性验收

- Outbox 失败重试可执行
- 通知消费失败后可重试
- 死信边界有清晰处理结果
- 处理中订单补查与补偿任务可执行
- Redis / MySQL / RocketMQ / Seata 故障注入结果有结论

## 5. 验收证据

- 至少一组支付成功全链路证据
- 至少一组支付失败全链路证据
- 至少一组退款成功全链路证据
- 至少一组 MQ 重试或消费证据
- 至少一组交易详情和审计查询证据

## 6. 通过标准

- 核心接口按约定返回成功或明确业务错误
- 数据库状态、MQ 行为、审计记录一致
- 没有“HTTP 成功但交易状态不一致”的悬空场景
- 验收中使用的配置来源、版本号和证据包可追溯
